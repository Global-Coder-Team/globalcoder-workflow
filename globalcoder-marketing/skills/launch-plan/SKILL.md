# Launch Plan

## Overview

Build a phased launch timeline with channels, milestones, and KPIs. The master coordination document for the entire go-to-market effort.

**Announce at start:** "I'm using the launch-plan skill to build the go-to-market timeline."

## Process

**Step 1: Context gathering**
- Read existing docs: positioning strategy, personas, brand identity
- Ask the output mode question (minimal vs full)

**Step 2: Scoping questions (one at a time)**

1. What's the target launch date? (specific date / "as soon as ready" / "exploring timeline")
2. What's the budget range? (none / small <$1k / moderate $1-10k / significant $10k+)
3. What channels are already established? (email list size, social followers, community, website traffic)
4. What's the launch style? (soft launch to small group / public beta / big-bang public launch)
5. Who's executing? (solo / small team / cross-functional team)

**Step 3: Build phased plan**

Present each phase separately for validation:

**Pre-launch (weeks/months before)**
- Build anticipation: waitlist, teaser content, early access program
- Partnerships and outreach: identify allies, press contacts, influencers
- Assets: landing page, email sequences, social content queue
- Internal: team alignment, support documentation

**Launch week**
- Coordinated push: announcement post, email blast, social campaign
- Community: Product Hunt / Hacker News / relevant communities (if applicable)
- Press/outreach: send to prepared contacts
- Monitor: track metrics in real-time, respond to feedback

**Post-launch (weeks after)**
- Sustain momentum: follow-up content, case studies, testimonials
- Iterate: adjust messaging based on early feedback
- Expand: new channels, partnerships, content

For each action: owner (if team), channel, timeline, success metric.

**Step 4: Write output**
- Write `docs/marketing/launch-plan.md`
- **Full mode adds:** launch checklist with checkboxes, milestone timeline as JSON, KPI tracking template

**Step 5: Handoff**
- Suggest `globalcoder-marketing:content-strategy` to flesh out content needs
- If globalcoder-workflow is available: "This plan has independent workstreams. Use `globalcoder-workflow:writing-plans` to break them into executable tasks."

## Key Principles

- **Phase by phase** - don't dump the whole timeline, validate incrementally
- **Realistic scope** - match plan ambition to team size and budget
- **Every action has a metric** - if you can't measure it, reconsider it
- **Channels over tactics** - focus on where personas actually are, not trendy platforms
